Contexte & choix technique clé

EXP-001 est la feature la plus différenciante du portfolio pour les recruteurs product. Le concept : uploader un screenshot d'interface et obtenir un audit UX structuré avec scoring, points de friction, et recommandations priorisées.

⚡ Décision critique — simulation vs API réelle : zéro appel Claude API. L'abonnement Claude Pro ne couvre pas les appels API (facturation séparée à l'usage). Simulation streaming identique à EXP-002 — trois audits hardcodés pour Airbnb, Google Analytics, LinkedIn. Les recruteurs voient le résultat et le process, pas si c'est "vrai".

Architecture de l'expérience

Colonne gauche — INPUT
  • Zone drag & drop avec bordure dashed → teal au hover
  • Preview base64 après upload
  • Badge statut dynamique (SCANNING / ANALYSING / AUDIT COMPLETE)
  • Chips "// Essayer avec :" — 3 fichiers pré-définis
  • Bouton NOUVELLE ANALYSE visible phase done
Colonne droite — OUTPUT
  • Phase idle : message d'attente
  • Phase scanning : 5 messages séquentiels toutes les 500ms
  • Phase streaming : texte caractère par caractère (3 chars / 20ms)
  • ## headers → Space Grotesk teal
  • Badges [CRITIQUE] [ATTENTION] [CORRECT] colorés
  • Progress bar CSS inline pour le score ██
  • Frictions F· en rouge, recommandations P· en cyan

Fix post-déploiement — chips UI

Constat immédiat au test visuel : les chips de suggestion ressemblaient à des champs de formulaire vides — inputs en colonne, couleur grise, bordure neutre. Un recruteur pourrait penser qu'il faut y saisir quelque chose.

Fix 01
Chips en colonne → chips horizontaux
AVANT : flex-direction column, gap 6px
→ ressemble à des inputs de formulaire
APRÈS : flex-wrap wrap, gap 8px
→ pattern chip tag standard ✓
Fix 02
Style neutre → couleur teal
AVANT : color #4a7a94, border #1a2a3a
→ invisible, sans signification
APRÈS : color #25e2cc, border teal 20%
→ cohérent avec le design system ✓
Fix 03
Label trop littéral
AVANT : "// Fichiers de démo utilisés :"
→ trahit explicitement la simulation
APRÈS : "// Essayer avec :"
→ guide sans exposer le mécanisme ✓
Fix 04
Workflow respecté
AVANT : correction directe dans le fichier
→ bypass du workflow Claude Code
APRÈS : correction via filesystem edit
→ documenté, traçable ✓
