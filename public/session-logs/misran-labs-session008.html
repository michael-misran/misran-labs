01

Décision stratégique — API vs Simulation

Avant de coder l'expérience, une décision critique s'est posée : utiliser la vraie Claude API avec streaming, ou simuler l'effet. La réponse était évidente une fois les coûts analysés.

Option A — Rejetée
Claude API réelle
~$0.04 par génération avec claude-haiku-4-5. Clé API exposée dans le bundle JS. Si le portfolio devient viral, les coûts explosent sans contrôle. Risque sécurité + financier inacceptable pour un portfolio public.
Option B — Retenue ✓
Streaming simulé
3 briefs pré-écrits de qualité professionnelle, affichés caractère par caractère. Coût : $0. Expérience visuelle identique. Le code montre la maîtrise du pattern — c'est ce qui compte pour un recruteur.

🎯 Note dans le code : // Simulation — voir le vrai appel API sur GitHub — transparence totale, positionnement professionnel qui montre la réflexion sur les coûts d'infrastructure.

02

EXP-002 — Interface & fonctionnement

La page est construite autour d'une interface 2 colonnes : INPUT à gauche, OUTPUT à droite. Le flux est immédiat et intuitif — aucune friction entre l'idée et le résultat.

État idle — Interface vide, 3 chips cliquables, bouton désactivé

Le bouton GÉNÉRER reste désactivé tant que le textarea est vide. Dès qu'un chip est cliqué ou du texte saisi, il s'active avec la couleur teal. UX state machine propre.

Chip sélectionné — 'Une app pour tracker ses lectures' actif, bouton prêt
Génération en cours — streaming caractère par caractère, curseur clignotant visible

✓ EXP-002 livrée et fonctionnelle. Streaming simulé à 18ms/3 chars, 3 briefs complets pré-écrits, fallback générique pour input libre, bouton COPIER après génération.

03

Architecture — Router générique des expériences

L'expérience a nécessité la création d'un système de routing extensible pour toutes les futures expériences du Lab.

ExperiencePage.jsx
Router générique
Lit le :slug dans l'URL et charge le bon composant. Une seule route /experience/:slug gère toutes les expériences actuelles et futures.
ProductBriefMachine.jsx
Composant autonome
Toute la logique dans src/experiences/ — séparé des pages. Pattern clair : chaque expérience est un composant indépendant, pluggé dans le router.
États UI
3 états distincts
idle → textarea vide, placeholder grisé. generating → streaming + curseur clignotant. done → brief complet + bouton COPIER.
Fallback
Brief générique
Si l'input ne correspond à aucun chip, un brief générique est utilisé. L'expérience fonctionne avec n'importe quelle saisie libre — pas seulement les 3 exemples.

3 états de l'interface

idle
Output vide avec texte placeholder grisé. Bouton désactivé. Invite à saisir ou cliquer un chip.
generating
Texte qui s'affiche caractère par caractère. Bouton passe à "GÉNÉRATION EN COURS...". Curseur | clignotant en fin de texte.
done
Brief complet affiché. Bouton COPIER apparaît. Sections ## colorées en teal, contenu en blanc cassé.
