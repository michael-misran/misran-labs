01

Pourquoi ce module

Principe établi depuis la session 008 : zéro surcoût sans décision explicite. Toutes les expériences du lab sont simulées pour cette raison. Mais dès qu'une vraie API sera branchée, il faut pouvoir voir en temps réel ce que ça coûte — tokens consommés, coût par appel, cumul session, alertes budget.

Ce module est la protection active contre les mauvaises surprises. Il transforme un risque invisible (facture API inattendue) en signal visible dans l'interface du lab.

⚡ Décision : construire le module maintenant avec des données simulées, plutôt qu'attendre d'avoir une vraie API. La structure est en place — quand une API sera branchée, seule la source des données change. L'interface reste identique.

02

Ce que le module affiche

Stats row — 4 métriques
  • SESSION COST — coût total en $ (rouge si budget dépassé)
  • TOKENS CONSOMMÉS — somme input + output
  • APPELS API — nombre de requêtes
  • COÛT MOYEN / APPEL — ratio coût/appels
Tableau des appels récents
  • Timestamp HH:MM:SS
  • Module source avec badge coloré
  • Modèle utilisé (sonnet teal / haiku vert / opus orange)
  • Tokens input + output
  • Coût calculé en temps réel
Budget alert bar
  • Barre de progression : coût / budget défini
  • Vert si < 80% du budget
  • Orange si 80-100%
  • Rouge si dépassé — "BUDGET DÉPASSÉ" visible
  • Seuil actuel : $0.10 par session
Répartition par modèle
  • 3 cards : claude-sonnet-4, haiku-4, opus-4
  • Appels / tokens / coût par modèle
  • Barre de proportion % du coût total
  • Permet d'identifier quel modèle coûte le plus
03

Tarifs Anthropic intégrés

Les coûts sont calculés avec les tarifs officiels Anthropic au 14.03.2026. La formule appliquée sur chaque appel :

coût = (inputTokens / 1_000_000 × prix_input) + (outputTokens / 1_000_000 × prix_output)

Modèle Input ($/1M tokens) Output ($/1M tokens) Usage recommandé
claude-sonnet-4 $3.00 $15.00 Défaut — équilibre qualité/coût
claude-haiku-4 $0.25 $1.25 Tâches simples — très économique
claude-opus-4 $15.00 $75.00 Tâches complexes — utiliser avec parcimonie

04

Quand une vraie API sera branchée

Le module est conçu pour être upgradé sans modifier l'interface. La seule chose qui change : la source des données.

Maintenant — simulation
  • MOCK_CALLS hardcodé dans le module
  • Calculs sur données statiques
  • Badge SIMULATION visible
  • Zéro appel réseau
Futur — vraie API
  • Store global (Context ou Zustand)
  • Chaque appel API → push dans le store
  • Module lit le store en temps réel
  • Badge LIVE remplace SIMULATION
