HomePage — identité visible

Problème identifié lors du bilan : la HomePage ne montrait pas qui tu es. Pas de nom, pas de titre, pas de phrase d'accroche. Un recruteur qui arrive sur la home ne sait pas en 10 secondes s'il est sur le portfolio d'un designer senior ou junior.

⚡ 3 modifications chirurgicales : ajout d'un label "MICHAEL MISRAN — PRODUCT DESIGNER × AI" en JetBrains Mono discret, remplacement du h1 générique par "Je conçois des produits. / Je les construis avec l'IA.", réécriture du sous-titre avec les outils concrets (Claude, Figma, Claude Code).

Avant
  • Hero anonyme — aucun nom visible
  • h1 générique sans différenciateur
  • Description vague "produits complets"
  • Un recruteur RH ne comprend pas le profil
Après
  • "MICHAEL MISRAN — PRODUCT DESIGNER × AI" visible immédiatement
  • h1 : "Je conçois des produits. Je les construis avec l'IA."
  • Sous-titre : Claude + Figma + Claude Code nommés
  • Profil lisible en 5 secondes par tout recruteur
02

/process — de générique à convaincant

La page /process listait 5 étapes correctes mais sans preuve. Un recruteur lisait "Je brief Claude Code avec des prompts structurés" sans jamais voir un seul prompt. C'est exactement le pattern à éviter : affirmer sans prouver.

Avant — assertions vides
  • "Je brief Claude Code avec des prompts structurés"
  • "Je critique, je corrige, j'itère"
  • Stack IA = tableau de logos sans contexte
  • Aucun exemple concret, aucune preuve
Après — preuves concrètes
  • Chaque étape a un exemple réel en mono
  • Étape 01 : brief EXP-001 verbatim
  • Étape 03 : prompt de fix chips verbatim
  • Étape 04 : commande git exacte
  • Étape 05 : structure WHAT/DECISION/PROMPT

Principe appliqué

→ Pattern: "Crée [fichier]. États : [liste]. Logique : [description].
   Layout : [colonnes]. Ne modifie pas [liste]."

→ Ex fix: "Les chips sont en colonne et ressemblent à des inputs.
   Corriger : flex-wrap wrap, couleur teal, border rgba teal.
   Ne rien toucher d'autre."

✓ La page /process est maintenant une démonstration de méthode, pas une liste de buzzwords. Un recruteur product comprend immédiatement comment tu penses et comment tu travailles.

03

Réflexions stratégiques de session

Refonte da Vinci — idée écartée

Idée impulsive évoquée en session : switcher le design Misran Labs vers un style "carnet de notes Leonardo da Vinci" (sépia, papier vieilli, annotations manuscrites). Garde-fou activé.

⚠ Rebranding complet = 10-15h de travail, perte du concept Jarvis, destruction de la cohérence existante. La vraie question était : "est-ce faisable ?" — oui techniquement (CSS textures + font Caveat + palette sépia). Noté en backlog pour un futur projet dédié.

i18n français/anglais — décision différée

Question soulevée : ajouter un toggle FR/EN. Pertinent pour les opportunités internationales, mais une internationalisation à moitié est pire que rien. Décision : garder le français comme langue principale, le mélange FR/EN dans les badges et labels techniques est crédible dans le monde tech. À faire après la première mission.

EXP-004 Design Token Extractor — en pause

La plus complexe techniquement — nécessite de la vision IA réelle (appel Claude API) pour extraire des tokens depuis une image. Impossible à simuler de façon convaincante. Maintenu en IN PROGRESS — une roadmap visible est un signal positif, pas un aveu d'échec.
