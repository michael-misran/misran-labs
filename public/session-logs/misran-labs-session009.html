Fix HomePage — Cards cliquables

En début de session, les 3 cards experiments de la HomePage n'étaient pas cliquables dans leur totalité — seul le lien texte "Voir l'expérience →" l'était. Fix appliqué avant de passer à la /design-system.

02

/design-system — Documentation vivante

Page générée entièrement en React — aucun outil intermédiaire. 5 sections : Colors, Typography, Components, Spacing, CTA. C'est le différenciateur fort du portfolio : un designer qui documente son DS en code.

Section COLORS — 9 swatches cliquables, copie du hex au clic

Chaque swatch est cliquable et copie la valeur hex dans le presse-papier. Un badge "COPIÉ ✓" apparaît 1.5s puis disparaît. useState par swatch pour gérer l'état indépendamment.

Section TYPOGRAPHY — 3 specimens : Space Grotesk / Inter / JetBrains Mono
Section COMPONENTS — Buttons et Status Badges avec code tokens à droite
Section COMPONENTS — Experiment Card avec hover state documenté
Section SPACING — Échelle visuelle 4px→80px + CTA final

04

Bugfixes /design-system — 3 corrections layout

Review visuelle post-livraison : 3 problèmes de layout identifiés et corrigés immédiatement.

Bug 01
Buttons — ligne cassée
✗ Le bouton "Non disponible" tombait en dessous des deux autres au lieu d'être aligné sur la même ligne.
✓ Fix : flexDirection: 'row' + flexWrap: 'wrap' sur le container buttons.
Bug 02
Colors — grille déséquilibrée
✗ La grille 5+4 laissait un grand espace vide en bas à droite, déséquilibrant la section.
✓ Fix : grid-template-columns: repeat(auto-fill, minmax(180px, 1fr)) — les 9 swatches s'adaptent proprement.
Bug 03
Experiment Card — trop large
✗ La card prenait toute la moitié gauche du layout avec beaucoup de vide autour.
✓ Fix : maxWidth: 320px sur le container de la card dans Components.
