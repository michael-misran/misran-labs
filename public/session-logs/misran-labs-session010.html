Fix final /design-system — Buttons

Dernier bug layout de la /design-system : les 3 boutons ne tenaient pas sur une ligne sans que le texte se wrap. Deux tentatives avant de trouver la bonne approche.

Tentative 1 : flexWrap: 'nowrap' — les boutons restaient sur une ligne mais le texte interne se wrappait car la colonne était trop étroite.

Tentative 2 : whiteSpace: 'nowrap' sur chaque bouton — résout le wrap texte mais les boutons dépassent la zone.

Solution retenue : layout vertical style Storybook — chaque bouton sur sa propre ligne avec un label (Primary / Secondary / Disabled). Plus propre, plus lisible, référence design system industry-standard.

Résultat final — layout vertical avec labels, propre et lisible

03

Architecture EXP-003

✓ Choix techniques
  • useState pour gérer l'accordéon — simple et suffisant
  • Un seul composant autonome dans src/experiences/
  • Données SESSIONS hardcodées dans le fichier — pas de fetch
  • Transition maxHeight 0→600px — CSS pure, pas de lib animation
  • Chevron ▼/▲ qui tourne au clic — feedback visuel immédiat
  • Router déjà prêt avant la livraison du composant
✓ Ce qui convainc un recruteur
  • 9 sessions documentées avec durées réelles (~18h total)
  • Décisions architecturales expliquées (Vercel vs Netlify, simulation vs API)
  • Prompts exacts visibles — transparence totale sur la méthode
  • Stats : 9 sessions / ~18h / 1 dev / 100% AI-assisted
  • CTA vers EXP-002 — flow naturel entre les expériences
  • Self-referential : l'expérience documente sa propre création

🎯 Pourquoi EXP-003 est le projet le plus fort du portfolio : il ne montre pas un résultat, il montre le processus. Un recruteur ne voit pas "voilà ce que j'ai fait" mais "voilà exactement comment je pense, décide et itère". C'est indupliquable.
