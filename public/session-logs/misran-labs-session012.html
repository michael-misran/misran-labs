01

Contexte & décision technique

Le formulaire /contact existait visuellement depuis la session 005 — terminal macOS, animations d'apparition, input ligne de commande. Mais le bouton SEND ne faisait rien : setSent(true) localement, aucun email réel envoyé.

La question soulevée : coder un backend email ou utiliser un service tiers ?

⚡ Décision : Formspree plutôt qu'un backend Node.js custom (Express + SMTP + Vercel function). Justification : le portfolio montre des compétences product et orchestration IA — pas du backend mail. Le temps économisé va sur EXP-001. Formspree est d'ailleurs le choix de la majorité des portfolios professionnels, y compris chez des devs seniors.

Deuxième décision : ajouter nom + email avant le message. La version initiale ne capturait qu'un message libre — impossible de recontacter l'expéditeur. Résolu via une UX séquentielle style terminal.

✓ Implémentation technique
  • useState step : 'name' | 'email' | 'message' | 'sent' | 'error'
  • handleEnter() async avec machine d'états séquentielle
  • fetch POST vers Formspree avec { name, email, message }
  • Bouton dynamique : SUIVANT → / ENVOYER / ENVOI...
  • Disabled pendant loading et après sent
  • Error state : message rouge + fallback email direct
✓ Ce que reçoit Formspree
  • Champ name : prénom/nom de l'expéditeur
  • Champ email : adresse de retour cliquable

  • Dashboard Formspree pour historique
03

Prompt Claude Code utilisé

Dans src/pages/ContactPage.jsx, modifie uniquement la fonction Terminal().

OBJECTIF : connecter le bouton SEND à Formspree + UX séquentielle nom → email → message.

1. Remplace useState sent par un state step : 'name' | 'email' | 'message' | 'sent' | 'error'
   Ajoute : const [name, setName] = useState('')
            const [email, setEmail] = useState('')
            const [loading, setLoading] = useState(false)

2. Remplace handleSend() par handleEnter() async — machine d'états :
   - step 'name'    → setName(input), setInput(''), setStep('email')
   - step 'email'   → setEmail(input), setInput(''), setStep('message')
   - step 'message' → fetch POST https://formspree.io/f/mjgarlpd
                       body: { name, email, message: input }
                       setStep(res.ok ? 'sent' : 'error')

3. JSX : afficher NOM et EMAIL saisis comme lignes de terminal au-dessus de l'input
4. Bouton : 'SUIVANT →' pour name/email, 'ENVOYER' pour message, 'ENVOI...' si loading
5. Messages : "> MESSAGE TRANSMIS ✓" en #25e2cc / "> TRANSMISSION ÉCHOUÉE ✗" en #ff4444

Ne modifie rien d'autre dans le fichier.

✓ Test validé — email reçu sur Gmail avec name: "Michael MISRAN", email: "misranmichael@gmail.com", message: "test message contact 002". L'adresse email apparaît en lien cliquable dans Formspree pour répondre directement.
