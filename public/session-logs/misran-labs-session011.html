Contexte de la session

Session courte, très ciblée. L'objectif : rendre les docs HTML de session accessibles depuis EXP-003 sans quitter le portfolio. Trois itérations successives pour atteindre la bonne UX — c'est exactement le type de process que le portfolio doit documenter.

Point de départ : EXP-003 Session Replay est live avec 9 cards accordéon. Les docs HTML existent en local mais sont invisibles pour le visiteur. La session 011 ferme cette lacune.

🎯 Décision d'architecture initiale : les HTML de session sont copiés dans public/session-logs/ et servis statiquement par Vite. Zéro backend, zéro fetch asynchrone — les fichiers sont disponibles instantanément à /session-logs/misran-labs-session001.html.

02

Feature : Session Log Viewer

Le principe : un lien // VOIR LE LOG COMPLET → apparaît dans chaque card accordéon ouverte. Il déclenche l'affichage du HTML correspondant sans navigation.

✓ Choix techniques
  • useState activeLog dans SessionReplay (null ou id session)
  • prop onOpenLog passée à chaque SessionCard
  • e.stopPropagation() pour ne pas toggle l'accordéon au clic
  • iframe src dynamique : /session-logs/misran-labs-session{id}.html
  • Overlay cliquable pour fermer (click en dehors)
  • Fichiers HTML dans public/ — servis par Vite sans config
✓ Ce que ça apporte au recruteur
  • Accès direct aux décisions architecturales session par session
  • Transparence totale sur la méthode de travail
  • Pattern UX propre : modale contextuelle, pas de navigation
  • Preuve d'itération : 3 versions successives documentées
  • Orchestration Claude Code visible — le prompt est dans le log

03

Bugs & itérations

Deux bugs identifiés immédiatement au test, une demande UX sur la troisième itération.

Bug 01 · Critique
Double zéro dans l'URL iframe
AVANT : /session-logs/misran-labs-session0{id}.html
→ génère session0001.html (inexistant)
APRÈS : /session-logs/misran-labs-session{id}.html
→ génère session001.html ✓
Bug 02 · Éditorial
Labels en anglais
AVANT : VIEW FULL LOG → / CLOSE
APRÈS : VOIR LE LOG COMPLET → / FERMER ✓

Itération UX — Modale plein écran → Modale overlay

La première version occupait tout l'écran sans overlay — le visiteur perdait le contexte et ne percevait pas la couche modale. Passage à une modale centrée 90vw × 85vh avec overlay blur(4px) cliquable pour fermer.

💡 Décision UX : l'overlay semi-transparent préserve le contexte de la page derrière. Le visiteur comprend immédiatement qu'il est dans une couche superposée et peut fermer en cliquant en dehors. Pattern standard, implémenté proprement sans librairie.

✓ Feature live — les 10 sessions sont accessibles depuis EXP-003 via modale overlay. Box-shadow cyan subtil sur la modale pour rester dans le design system.
