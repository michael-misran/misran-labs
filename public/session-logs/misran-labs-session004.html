01

Le prompt de correction

Un prompt court et ciblé — 4 corrections numérotées, pas de reformulation complète. Le but est de ne toucher que ce qui doit changer sans risquer de casser ce qui fonctionne.

Prompt envoyé à Claude Code
Dans HomePage.jsx, fais ces corrections : 1. Ajoute une border visible sur les cartes experiment au repos : border: '1px solid #1a2a3a' hover : border-color '#00d4ff' + box-shadow '0 0 20px rgba(0,212,255,0.15)' 2. Corrige la description de Session Replay (EXP-003) : "Documentation narrative d'un build en live avec Claude Code. Prompts, décisions, itérations — le making-of complet." 3. Ajoute la grille blueprint sur le wrapper principal : backgroundImage avec deux linear-gradient de couleur #1a2a3a, background-size 40px 40px, opacity 0.4 4. Réduis le padding entre la section experiments et le CTA de moitié.

🎯 Pattern correction : Pour les itérations, toujours numéroter les corrections et rester chirurgical. Un prompt de correction diffus ("améliore le design") risque de déclencher une réécriture complète du fichier et d'introduire des régressions.
