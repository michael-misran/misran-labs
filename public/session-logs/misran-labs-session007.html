Bug : Vercel 404 sur accès direct aux routes

Après le push de ContactPage, l'accès direct à michaelmisran.com/contact retourne une erreur Vercel 404. Le site semblait cassé alors que le code était correct.

Chronologie du diagnostic

Symptôme
404 NOT_FOUND sur /contact — Vercel ne trouve pas la page
Hypothèse 1 — Rejetée
Route manquante dans App.jsx — vérification via MCP Filesystem : /contact est bien déclarée
Hypothèse 2 — Rejetée
Erreur dans ContactPage.jsx — lecture des 30 premières et dernières lignes : code propre, aucune erreur de syntaxe
Cause réelle identifiée
vercel.json inexistant sur le repo. Vercel cherche un fichier physique à chaque route directe. Sans rewrite SPA, toute URL tapée directement retourne 404.
Résolution
Création de vercel.json avec rewrite source: "/(.*)" → destination: "/index.html". Claude Code ajoute aussi la route /experience/:slug et un catch-all dans App.jsx.

💡 Apprentissage clé : Ce bug existait depuis le début — /lab et /process n'avaient jamais été testées en accès direct. La navigation par liens internes fonctionne car React Router gère côté client. L'accès direct (URL bar, partage de lien) nécessite le rewrite Vercel. Ce fichier doit être créé dès le setup initial.

Fix appliqué

// vercel.json
{
  "rewrites": [
    { "source": "/(.*)", "destination": "/index.html" }
  ]
}
// App.jsx — ajouts de Claude Code
import { Navigate } from 'react-router-dom'
<Route path="/experience/:slug" element={<LabPage />} />
<Route path="*" element={<Navigate to="/" replace />} />

MCP Filesystem — Accès au repo local débloqué

Pour diagnostiquer le bug sans que Michael copie-colle le code manuellement, l'accès MCP Filesystem a été étendu au dossier /Users/michaelmisran/misran-labs. Claude peut maintenant lire et écrire directement dans le repo.

Configuration

// ~/Library/Application Support/Claude/claude_desktop_config.json
{
  "mcpServers": {
    "filesystem": {
      "command": "/Users/michaelmisran/.npm-global/bin/mcp-server-filesystem",
      "args": [
        "/Users/michaelmisran/Documents",
        "/Users/michaelmisran/Downloads",
        "/Users/michaelmisran/misran-labs"   // ← ajouté
      ]
    }
  }
}
Ce que ça change
Diagnostic direct sans copier-coller
Claude lit App.jsx, ContactPage.jsx, vérifie l'existence de vercel.json et crée le fichier directement — sans que Michael touche au terminal.
Fichiers créés directement
vercel.json écrit par Claude
Claude a créé /Users/michaelmisran/misran-labs/vercel.json directement via MCP Filesystem. Michael n'avait qu'à pusher avec Claude Code.
Workflow amélioré
Debugging autonome
Pour les prochains bugs : Claude lit le fichier concerné, identifie le problème, écrit le fix, Michael push. Boucle complète sans friction.
Limite à garder en tête
Claude ne peut pas pusher
MCP Filesystem donne accès aux fichiers locaux, pas à git. Le push reste dans Claude Code. Michael reste le seul à déclencher les déploiements.
