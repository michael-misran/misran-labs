01

Le pivot — de portfolio à laboratoire

La session commence par une prise de conscience claire : "Je t'utilise comme Jarvis". Le mot portfolio est trop restrictif. Ce qui est construit ici est un environnement de travail personnel qui évolue en continu — un vrai laboratoire scientifique, pas une vitrine statique.

⚡ Décision radicale : supprimer toutes les pages de la V1 et reconstruire un shell applicatif de type interface Jarvis. Pas de branche, pas de sauvegarde — on avance. Même repo, même déploiement Vercel, expériences existantes conservées et migrées en modules.

V1 — Site portfolio
  • Navigation = liens entre pages
  • Chaque page recharge le layout
  • Pas de contexte global ni d'état persistant
  • Design "site web" classique
  • Objectif : montrer aux recruteurs
V2 — Shell applicatif
  • Navigation = switching de modules
  • Frame persistante, seul le contenu change
  • État global : module actif, historique, session
  • Design "application desktop Jarvis"
  • Objectif : laboratoire personnel évolutif

La philosophie change également : "Si un projet ne me plaît pas je le jette" — comme Tony Stark avec ses combinaisons. Le lab est fait pour expérimenter en code réel, pas pour valider en Figma. Le code React est le Figma.

03

Widgets live — zéro surcoût

Principe maintenu depuis le début : aucun service payant sans décision explicite. Les 4 widgets de la sidebar respectent ce principe.

ClockWidget — natif JS
  • new Date() — zéro API, zéro dépendance
  • HH:MM en grand + :SS en petit
  • Date dessous en Inter 11px
  • setInterval 1000ms + clearInterval unmount
WeatherWidget — Open-Meteo
  • Gratuit illimité, zéro clé API requise
  • Coords Paris : 48.8566°N / 2.3522°E
  • weathercode WMO → emoji + description FR
  • Refresh toutes les 10 minutes
  • Résultat validé : 9°C ☀️ Ensoleillé Paris
UptimeWidget
  • startTime stocké en sessionStorage au mount
  • Format HH:MM:SS mis à jour chaque seconde
  • Reset à chaque nouvelle session navigateur
ActiveModuleWidget
  • Module actif : icône + label en temps réel
  • Historique des 3 derniers modules
  • State géré dans App.jsx, descendu via props
